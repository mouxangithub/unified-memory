# Unified Memory v3.8.0 Release Notes

**Release Date**: 2026-03-30  
**Version**: 3.8.0  
**Total Tools**: 112 (97 + 15 new)

---

## 🚀 Major Features

### 1. WAL Protocol (Write-Ahead Log)

Complete Write-Ahead Log implementation for data durability and crash recovery.

**New Tools**:
- `memory_wal_write` - Write entry to WAL with checksum
- `memory_wal_replay` - Replay WAL entries for crash recovery
- `memory_wal_status` - Get WAL status and statistics
- `memory_wal_truncate` - Truncate WAL after successful commit
- `memory_wal_export` - Export WAL for backup
- `memory_wal_import` - Import WAL from backup

**Features**:
- ✅ Checksum verification for data integrity
- ✅ Automatic crash recovery on startup
- ✅ Export/import for backup and migration
- ✅ Backward compatible with existing storage.js

**Usage Example**:
```bash
# Write to WAL
mcporter call unified-memory.memory_wal_write(
  operation: "insert",
  collection: "memories",
  data: { text: "Important memory", importance: 0.9 }
)

# Check WAL status
mcporter call unified-memory.memory_wal_status()

# Export for backup
mcporter call unified-memory.memory_wal_export()
```

---

### 2. Evidence Chain Mechanism

Track the source and evolution of memories through evidence chains.

**New Tools**:
- `memory_evidence_add` - Add evidence to memory's chain
- `memory_evidence_get` - Get evidence chain for a memory
- `memory_evidence_find_by_type` - Find memories by evidence type
- `memory_evidence_find_by_source` - Find memories by source ID
- `memory_evidence_stats` - Get evidence statistics

**Features**:
- ✅ Source tracking (transcript/message/manual/inference)
- ✅ Confidence scoring (0-1)
- ✅ Evidence chain visualization
- ✅ Source-based filtering and search

**Usage Example**:
```bash
# Add evidence
mcporter call unified-memory.memory_evidence_add(
  memoryId: "mem_123",
  type: "transcript",
  sourceId: "msg_456",
  confidence: 0.95,
  context: "User mentioned preference for dark mode"
)

# Get evidence stats
mcporter call unified-memory.memory_evidence_stats()
```

---

### 3. Auto Organization

Automatic memory tiering and compression across HOT/WARM/COLD tiers.

**New Tools**:
- `memory_organize` - Organize memories across tiers
- `memory_compress_tier` - Compress memories in a specific tier
- `memory_archive_old` - Archive memories older than threshold
- `memory_tier_stats` - Get tier statistics
- `memory_full_organize` - Run full organization (organize + compress + archive)

**Features**:
- ✅ Automatic tier migration based on age and importance
- ✅ Memory compression (summarization)
- ✅ Archive old memories (365+ days)
- ✅ Tier-specific compression ratios
- ✅ Statistics and monitoring

**Tier Configuration**:
- **HOT**: 7 days max age, 10 memories, 50% compression
- **WARM**: 30 days max age, 50 memories, 30% compression
- **COLD**: 365 days max age, 1000 memories, 10% compression

**Usage Example**:
```bash
# Full organization
mcporter call unified-memory.memory_full_organize()

# Get tier stats
mcporter call unified-memory.memory_tier_stats()

# Archive old memories
mcporter call unified-memory.memory_archive_old(thresholdDays: 365)
```

## 📝 Migration Guide

### From v3.7.0 to v3.8.0

**No breaking changes** - All existing tools remain functional.

**Recommended Actions**:
1. Enable WAL protocol for critical data
2. Start tracking evidence for important memories
3. Run `memory_full_organize` to optimize memory tiers

**Configuration** (Optional):
```bash
# Enable WAL (enabled by default)
export ENABLE_WAL=true

# Configure evidence tracking
export EVIDENCE_TRACKING=true

# Configure auto-organization schedule
export AUTO_ORGANIZE_INTERVAL=86400  # 24 hours
```
