# QMD 风格搜索

基于 QMD 架构的本地优先搜索系统，支持三层可选增强。

## 架构

```
用户查询
    ↓
┌─────────────────────────────────────┐
│  Layer 1: BM25（默认，0 Token）      │
│  纯关键词匹配，完全本地              │
└─────────────────────────────────────┘
    ↓ 可选
┌─────────────────────────────────────┐
│  Layer 2: 向量搜索（~100 Token）     │
│  语义搜索 + RRF 融合                │
│  需要本地 Ollama                    │
└─────────────────────────────────────┘
    ↓ 可选
┌─────────────────────────────────────┐
│  Layer 3: LLM 重排（额外 Token）     │
│  智能排序，提升相关性               │
│  需要配置 LLM 模型                  │
└─────────────────────────────────────┘
```

## 配置

```bash
# 查看当前配置
python3 memory_qmd_search.py config

# 设置配置
python3 memory_qmd_search.py config --set-config '{
  "use_vector": true,
  "use_llm_rerank": false,
  "llm_model": "qwen2.5:0.5b"
}'
```

### 配置项

| 配置 | 说明 | 默认值 |
|------|------|--------|
| `use_vector` | 是否启用向量搜索 | `true` |
| `use_llm_rerank` | 是否启用 LLM 重排 | `false` |
| `llm_model` | 重排用的模型 | `qwen2.5:0.5b` |

## 使用

```bash
# 自动模式（根据配置选择）
memq search -q "查询内容"

# 纯 BM25（0 Token）
memq search -q "查询" -m bm25

# 混合搜索（BM25 + 向量）
memq search -q "查询" -m hybrid

# 启用 LLM 重排
memq search -q "查询" --rerank

# 临时关闭向量
memq search -q "查询" --no-vector
```

## Token 消耗

| 模式 | LLM 调用 | Token | 适用场景 |
|------|---------|-------|---------|
| BM25 | 0 | **0** | 精确关键词 |
| 向量 | 1 次 embedding | ~100 | 语义搜索 |
| 混合 | 1 次 embedding | ~100 | 综合最佳 |
| +重排 | +1 次 LLM | +300 | 最高质量 |

## 与 QMD 对比

| 功能 | unified-memory | QMD |
|------|---------------|-----|
| BM25 搜索 | ✅ | ✅ |
| 向量搜索 | ✅ | ✅ |
| RRF 融合 | ✅ | ✅ |
| 本地重排 | ✅ 可选 | ✅ 默认 |
| 片段返回 | ✅ | ✅ |
| Agent 记忆管理 | ✅ | ❌ |
| 用户画像 | ✅ | ❌ |
| 分层配置 | ✅ | ❌ |

## 优势

1. **本地优先**：BM25 默认启用，0 Token
2. **渐进增强**：可选向量、LLM 重排
3. **灵活配置**：配置文件 + 命令行参数
4. **Agent 集成**：保留记忆管理功能
