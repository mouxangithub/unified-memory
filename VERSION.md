## 语义化版本控制 (SemVer)

```
MAJOR.MINOR.PATCH
  │     │     │
  │     │     └── 补丁版本：Bug 修复、小优化
  │     └──────── 次版本：新功能、向后兼容
  └──────────────── 主版本：重大架构变更、不兼容更新
```

## 为什么用 0.0.7 而不是 v7.0？

| 问题 | v7.0 | 0.0.7 |
|------|------|-------|
| 跨度过大 | ❌ 直接跳到 7.0 | ✅ 第7次迭代 |
| 语义不清 | ❌ 不明确变化程度 | ✅ PATCH 级更新 |
| 用户预期 | ❌ 期望重大变化 | ✅ 期望渐进改进 |
| 版本规划 | ❌ 后续难升级 | ✅ 有清晰路线 |

## 版本号规则

### PATCH (0.0.x)
- Bug 修复
- 性能优化
- 小功能改进
- 文档更新

### MINOR (0.x.0)
- 新功能模块
- API 扩展
- 向后兼容的改进

### MAJOR (x.0.0)
- 架构重构
- 不兼容的 API 变更
- 重大功能变化
